# 구현 플랜 개요

## 컨텍스트 윈도우 운영 규칙

- 각 스텝 완료 후: 다음 스텝 진행 가능 여부 판단
- 윈도우 소진 임박 시: `HANDOFF_{STEP}.md` 작성 후 중단, 사용자에게 알림
- 새 세션 시작 시: 해당 HANDOFF 파일 읽고 이어서 진행

## 스텝 목록

| 스텝 | 내용 | 파일 |
|------|------|------|
| 01 | DB 모델 + Alembic 마이그레이션 확정 | STEP_01_db_setup.md |
| 02 | 종목코드 초기 로드 (KRX) | STEP_02_stock_codes.md |
| 03 | Telethon 수집기 (백필 + 리스너) | STEP_03_collector.md |
| 04 | 메시지 파서 (@repostory123 + 정규화) | STEP_04_parser.md |
| 05 | DB upsert + 중복 제거 | STEP_05_storage.md |
| 06 | PDF 아카이빙 | STEP_06_pdf.md |
| 07 | 추가 채널 확장 | STEP_07_channels.md |

## 각 스텝 구성

- **목표**: 이 스텝에서 달성할 것
- **사전 조건**: 이전 스텝 완료 여부 체크
- **구현 대상 파일**
- **테스트 코드**
- **검증 체크리스트**: 완료 기준
- **다음 스텝 진입 조건**
